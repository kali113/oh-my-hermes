# oh-brain

Use this when the task depends on durable user/project knowledge, previous decisions, private notes, or repeated workflows.

1. Check Hermes memory first when the answer may depend on user preferences or prior context.
2. If `gbrain` is available, use it for durable lookup, ingestion, or synthesis.
3. Do not write secrets, access tokens, private identifiers, or raw logs into public reports.
4. When updating memory, prefer concise facts, durable preferences, and links to evidence.
