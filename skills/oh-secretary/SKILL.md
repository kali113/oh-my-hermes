# oh-secretary

Use this when the user wants personal-agent behavior: briefings, task tracking, reminders, decisions, follow-ups, and coordination.

1. Start with the private secretary state under `~/.oh-hermes/secretary`.
2. Capture durable tasks, preferences, and decisions with `oh-hermes secretary capture`.
3. Use `oh-hermes secretary brief` for a daily operating summary.
4. Ask before sending messages, changing calendar/email data, spending money, or publishing anything.
5. Keep private account data out of the publishable repo.
6. When a workflow repeats, turn it into a checklist or skill proposal so god-mode can improve it later.
